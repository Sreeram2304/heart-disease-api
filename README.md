# CardioScan — Heart Disease Prediction App

A full-stack ML web application that predicts heart disease risk from patient vitals.

**Stack**: FastAPI · scikit-learn · SQLite/PostgreSQL · JWT Auth · Jinja2 · Chart.js · Docker

[![CI](https://github.com/YOUR_USERNAME/heart-disease-api/actions/workflows/ci.yml/badge.svg)](https://github.com/YOUR_USERNAME/heart-disease-api/actions)

---

## Features

| Feature | Details |
|---|---|
| ML inference | Random Forest, 13 features, ~84% accuracy |
| Auth | JWT register/login, bcrypt passwords |
| Database | SQLite (dev) / PostgreSQL (prod) via SQLAlchemy |
| Frontend | Prediction form + analytics dashboard |
| Charts | Outcome donut, probability trend, model metrics |
| History | Per-user prediction log |
| CI/CD | GitHub Actions → pytest → Docker build |

│   ├── index.html            # Prediction form (auth + vitals input)

## Quick start

```bash
# Install
pip install -r requirements.txt

# Train model
python train.py

# Run server
uvicorn main:app --reload

# Open in browser
open http://localhost:8000
```

## API endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/api/auth/register` | — | Create account |
| POST | `/api/auth/login` | — | Get JWT token |
| GET | `/api/auth/me` | JWT | Current user info |
| POST | `/api/predict` | JWT | Run prediction + save to DB |
| GET | `/api/history` | JWT | User's prediction history |
| GET | `/api/stats` | JWT | Aggregated stats + model metrics |
| GET | `/health` | — | Liveness check |

## Deploy to Render (free)

1. Push to GitHub
2. Render → New Web Service → Docker → port `8000`
3. Set env vars: `SECRET_KEY=your-secret`, optionally `DATABASE_URL` for Postgres
4. Deploy — live URL goes on your resume
